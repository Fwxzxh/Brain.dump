Data Structures

1 What is a Data Structure? (DS)

  • Una manera de organizar datos para que puedan ser usados de manera efectiva.

2 Why Data Structures?

  • Son esenciales para crear algoritmos rápidos y eficientes.
  • Ayudan a manejar y organizar datos.
  • Hacen que el código se vea limpio y sea fácil de entender.

3 Abstact Data Type (ADT)

Son una abstracción de una estructura de datos que provee solo la interfaz a la que una estructura de datos deberia de adherirse.

Esta no da ningún detalle especifico sobre como algo debería de ser implementado.

Si tuviera que llegar del punto A al punto B el ADT seria la tarea de llegar de “A a B” y la estuctura de datos sería el medio para llegar de “A a B”.

Abstraction (ADT) Implementation (DS)
List Dynamic Array, Linked List
Queue Linked List, Based Queue, Array based Queue, Stack based Queue
Map Tree Map, Hash Map/Hash Table
Vehicle Golf Cart, Bycycle, Smart Car

4 Computational Complexity Anlalysis

Se trata de preguntarnos dos preguntas: * ¿Cuánto tiempo este algoritmo tarda en terminar? * ¿Cuánto espacio necesita este algoritmo para trabajar?

5 Big-O Notation

Esta notación nos da un tope de la complejidad en el peor caso, ayudando a cuantificar el rendimiento cuando el tamaño de la entrada de datos se hace arbitrariamente grande.

5.1 Niveles de complejidad

Siendo \(n\) el tamaño de la entrada, La complejidad va de la de menor a mayor.

  1. Tiempo Constante: \(O(1)\).
  2. Tiempo Logarítmico: \(O(log(n))\).
  3. Tiempo Linear: \(O(n)\).
  4. Tiempo Linearitmic: \(O(n*log(n)\).
  5. Tiempo Cuadrado: \(O(n^2)\).
  6. Tiempo Cúbico: \(O(n^3)\).
  7. Tiempo Exponencial: \(O(b^n),b>1\).
  8. Tiempo Factorial: \(O(n!)\).

5.2 Propiedades de Big-O

\begin{aligned} O(n+c)&= \mathbf{O(n)} \\ O(cn)&= \mathbf{O(n)}, c>0 \end{aligned}

Sea f una función que describe el tiempo de ejecución de un algoritmo para una entrada de tamaño \(n\):

\begin{aligned} f(n) &= 7log(n)^3 + 15n^2 + 2n^3 + 8 \\ O(f(n)) &= \mathbf{O(n^3)} \end{aligned}

5.2.1 Ejemplos Big-O

  1. Tiempo Constante

    El siguiente ejemplo corre en tiempo constante: \(0(1)\) 

      a := 1      
  b := 1      
  c := a + 5*b    

      i := 0
  While i < do
  i = 1+1
  2. Tiempo Lineal

    Los siguientes ejemplos corren en tiempo lineal: \(\mathbf{O(n)}\) 

      i := 0
  While i < n Do
      i = i + 1

    Expresado matemáticamente

    \begin{aligned} f(n) &= n \\ O(f(n)) &= \mathbf{O(n)} \end{aligned}

    Siendo \(n\) el valor mayor posible llegaremos en \(n\) iteraciones. 

      i := 0
  While i < n Do
      i = i + 3

    Expresado matemáticamente

    \begin{aligned} f(n) &= n/3 \\ O(f(n)) &= \mathbf{O(n)} \end{aligned}

    Siendo \(n\) el valor mayor posible llegaremos a \(n\) en una tercera parte de las iteraciones del ejemplo anterior.

  3. Tiempo Cuadrático

    Ambos de los siguientes ejemplos corren en tiempo cuadrático.

    El primero puede que sea ovbio desde que \(n\) trabajos hechos en \(n\) tiempos es \(n*n = \mathbf{O(n^2)}\) Pero el segundo.. 

      For (i := 0 ; i < n; i = i + 1)
      For (j := 0 ; j < n; j = j + 1)

      f(n) = n* n = n^2, O(f(n)) = \mathbf{O(n^2)}

      For (i := 0 ; i < n ; i = i + 1)
      For (j := i ; j < n ; j = j = 1)

    En el segundo For. i toma valores desde \(`[0,n)`\) la cantidad de ciclos que hace es directamente determinada por que valor i es.

    Si i=0 hace \(`n`\) iteraciones, si i=1 hacemos \(`n-1`\) iteraciones, si hacemos i=2, hacemos \(`n-2`\) iteraciones, etc…

    Esto se convierte en \(n + (n-1) + (n-2) + (n-3) + \dots + 3 + 2 + 1\) Esta serie se convierte en: \(n(n-1)/2\) así que.

    \begin{displaymath} O(n(n+1)/2) = O(n^2/2 + n/2) = \mathbf{O(n²)} \end{displaymath}
  4. Tiempo Logarítmico

    Supongamos que tenemos un array ordenado y queremos encontrar el índice de un valor particular, Si este existe.

    ¿Cuál es la complejidad en el tiempo del sigiente algoritmo? 

      low := 0 
  high := n-1 
  While low <= high Do
      mid := (low + high) / 2

      If array[mid] == value: return mid
      Else If array[mid] < value: lo = mid + 1
      Else If array[mid] > value: hi = mid - 1

  return -1 //value not found

    Respuesta: \(O(log_2(n)) = \mathbf{log(n))}\).

  5. Otros ejemplos

    ¿Cuál es la complejidad en el tiempo del sigiente algoritmo? 

      i := 0
  While i < n Do
      j = 0
      While j < 3*n Do
          j = j + 1
      j = 0 
      While j < 2 * n Do
          j = j + 1
      i = i + 1

    Representado matemáticamente

    \begin{aligned} f(n) &= n \cdot (3n + 2n) = 5n^2 \\ O(f(n)) &= \mathbf{O(n^2)} \end{aligned}

    ¿Cuál es la complejidad en el tiempo del sigiente algoritmo? 

      i := 0
  While i < 3 * n Do
      j := 10
      While j <= 40 Do
          j = j + 1
      j = 0
      While j < n*n*n Do
          j = j + 2
      i = i + 1
    \begin{aligned} f(n) &= 3n \cdot (40 + n^3 / 2) = 3n/40 + 3n^4/2 \\ O(f(n)) &= \mathbf{O(n^4)} \end{aligned}
